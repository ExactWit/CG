# 论文笔记
>[!problem]
>传统曲面 CNN（MeshCNN、HSN、Geodesic CNN 等）的核心矛盾是：卷积核必须和网格拓扑 / 局部参数化绑定，导致过拟合连通性、跨采样泛化差、实现复杂。

>[!remark]
>三角网格中顶点的邻居数目是不确定的，曲面mesh不具备天然的、规则的“卷积窗口”。
>![[不定邻居.png]]
## 三类mesh卷积方法
>[!proposition]
>**基于边 / 面拓扑的卷积（代表：MeshCNN）**
>MeshCNN 直接把**边**作为卷积的基本单元（类似图像的像素）：
>- 每条边的感受野固定为它的 4 条相邻边（共享两个三角形面的邻边）；
>- 卷积核就是 1 个中心边权重 + 4 个邻边权重，直接做加权求和；
>- 池化通过**边折叠**实现，每池化一次就动态更新整个网格的拓扑连接关系。

>[!remark]
>该类方法**完全绑定网格的边拓扑结构**。卷积核的输入结构、邻接关系、特征顺序，全部由三角剖分的连接方式决定。

这带来了三个问题：
1. **严重过拟合连通性**
    
    网络学到的本质是 “在这张特定的边连接图上怎么做特征聚合”，而不是形状本身的几何规律。
    
    比如 FAUST 数据集所有模型都共享完全相同的拓扑（顶点数、边连接全一样，只是坐标不同），ACSCNN 在上面精度很高；但只要把测试集重新三角剖分，边的数量和连接关系一变，训练好的卷积核直接失效，因为输入的 “结构” 都不对了（Table5,Fig10）。
    ![[remesh.png]]
    
2. **跨采样泛化为零**
    
    不同分辨率、不同剖分的网格，边的数量、邻接关系、排列顺序没有对应关系，模型根本无法直接迁移。这也是为什么这类方法必须把所有数据先简化到统一的低分辨率网格才能训练。
    
3. **实现复杂且脆弱**
    
    边折叠池化需要动态维护拓扑关系，每一层都要重新计算邻接、更新特征映射，工程实现非常繁琐；而且对网格质量要求高，非流形、退化三角形都会导致边折叠失败。

>[!propisition]
**基于局部参数化的测地线卷积（代表：GCNN、ACNN、MoNet）**
>该类方法把曲面一小块 “展平”，就能套用 2D 卷积了。
>- 在每个顶点处，取一个测地邻域（测地距离范围内的所有点）；
>- 通过**对数映射**把邻域内的点投影到该点的切平面上，得到平面极坐标；
>- 在切平面这个 “二维图片” 上执行标准卷积。

>[!remark]
>该类方法**绑定了局部参数化的全部细节**：测地邻域的选取方式、对数映射的计算、切平面坐标系的朝向，全部依赖当前网格的离散结构。

这也带来了三个问题：
1. **邻域采样不稳定**
    
    同一点在不同三角剖分下，测地邻域内的顶点数量、空间分布都不一样，展平后的 “采样点” 对不齐，卷积核权重无法共享。就像两张分辨率不同的图片，像素格对不上，直接用同一个卷积核必然失效。
    
2. **方向歧义无法根除**
    
    切平面没有天然的 “正北方向”，角度坐标的原点是人为随便选的。不同网格、不同顶点的局部坐标系朝向不一致，卷积核的方向语义不统一。后续大量工作（如 HSN）就是为了解决这个方向问题，但又引入了更复杂的设计。
    
3. **计算昂贵且鲁棒性差**
    
    逐顶点计算测地线和对数映射本身就是几何处理的重活，高曲率区域还会出现映射畸变；真实扫描数据有噪声、孔洞时，测地距离计算很容易出错，整个网络跟着崩。


>[!proposition]
>**基于切向量场的等变卷积（代表：HSN、FCNet）**
为了解决切平面方向不一致的问题，这类方法直接设计**旋转等变的卷积核**：
>- 特征定义在切向量空间上，卷积核用调和基 / 复值函数构造；
>- 通过平行传输把邻点特征对齐到中心顶点的切平面，保证切平面旋转时，特征也能对应地等变变换。

>[!remark]
>该类方法**绑定了局部标架和平行传输路径**。标架怎么选、平行传输沿哪条边走，都由网格的拓扑连接决定。

带来的三个问题：

1. **等变性只在理想网格上成立**
    
    平行传输的精度高度依赖网格质量和连接方式。不同三角剖分下，同一段路径传输后的向量方向有偏差，理论上的等变性在实际离散网格上被破坏，跨采样泛化依然很差 ——DiffusionNet 论文 Table 5 里 HSN 重网格化后误差从 9.57 暴涨到 20 以上，就是明证。
    
2. **实现复杂度极高**
    
    要处理复值特征、调和分解、协变导数，还要保证每一层的等变性不丢失，数学和工程双重复杂；调试难度大，对输入网格的流形性、朝向一致性要求苛刻。
    
3. **计算成本高，难以规模化**
    
    向量场运算和平行传输很难纯矩阵化加速，只能在低分辨率简化网格上运行，无法直接处理原始扫描级别的高分辨率网格。
