# 问题
![[BRepCLIP_fig1.png]]
>[!problem]
>现有3D多模态方法通常使用**点云表示**，目前没有多模态方法能有效处理**CAD模型**的**原生**BRep结构：
>- 基于**点云表示**的方法（ULIP-2 CVPR 2024）将精确的边界表示退化为无序坐标集，丢失解析曲面类型、曲线基元、拓扑邻接关系；
>- **网格表示**、**体素表示**因其固有缺陷，在3D多模态方法中较少出现；
>- 现有BRep学习方法（如UV-Net、BRepNet）只关注CAD数据本身的预训练，未对齐语言/图像模态，无法做**开放词汇检索**。

>[!remark]
>在**3D多模态学习**的工作中**点云表示**是主流。
>**体素表示**因其内存占用高、分辨率受限的特性，直接出局；而主流工作较少出现**网格表示**的主要原因是缺乏**典范网格化方法**，这导致了Mesh表示具有以下缺陷：
>- 不同样本的边数、面数差异极大，批次化计算性能浪费大（padding、masking）；
>- 不同复杂度模型的恰当Mesh化面数难以被同一固定数目满足；
>- 作为Graph的连通信息无法与几何特征一一对应（同一个圆柱可以对应无数种图结构）
>针对CAD领域，点云、网格、体素三种表示方法无法满足参数化编辑、制造可行性等任务需求，因此亟需基于BRep的学习方法。
>
>而在其他3D任务中，最优表示未必是BRep，如**3D生成**中对于扩散模型的路径依赖使得**体素**表示作为2D像素的扩展更能兼容已有方法；基于有限元方法的**物理仿真**需要**体网格**......

# 动机
>[!motivation]
>工程师很少从头设计零件，**检索**已有零件进行**复用**效率更高，这要求：
>- 开放词汇检索（需要突破固定类别）
>- 细粒度几何判别（孔数、边类型、曲面类型等）
>- 跨模态对齐（文本/图像$\leftrightarrow$CAD模型）
>现有主流3D多模态预训练主要基于点云表示，BRep结构学习通常是单一模态，鲜有工作将BRep原生结构与文本、图像进行对比学习

# 方法
![[VQ-VAE.png]]
>[!solution]
>在输入数据上，该方法分别从面、边上采样面点与边点，记录其坐标、所属面/边id及其质心坐标、所属面/边类型标签（柱面、环面、圆弧、B-spline等）
>
>**BRepCLIP**的核心部件是用于编码面、边语义信息的**Hybrid dual-dVAE tokenization**，实现上使用PointNet编码面，1维卷积编码边，再用两个独立的VQ-VAE（重建任务）得到基元表示。
>
>在图文表示上，使用预训练CLIP编码CAD模型的多视角图像和文本描述，同BRep语义特征一起进入Transformer（L12,d384,h6）进行训练。
>
>最终得到一个图文CAD检索模型，输入图文描述，为库中所有CAD模型打一个相关性得分。
![[backbone.png]]

>[!remark]
>事实上，BRepCLIP并未直接使用BRep拓扑图和参数方程，仅使用**半结构化点云**来获取一定的几何语义。
>
>BRepCLIP的基本功能是给出图、文、CAD模型的语义相似度得分。除了满足工程师的**检索-复用需求**，还可以作为一个CAD生成模型（DreamCAD 2026，同团队工作）的**质量评估器**，在人工评估的介入下实现半自动“数据飞轮”：评判越准，训练数据越好，生成质量越好，评判越准...
>
>当然，这个愿景的实现会遇到极多的阻碍，需要极大的资源投入，这里不展开讨论。

# 贡献与局限
>[!contribution]
>- 首次BRep原生多模态对比学习，填补BRep结构$\leftrightarrow$文本/图像对齐的空白
>- SOTA检索、零样本分类性能，ABC上Top-1检索比OpenShape（NIPS 2023）相对高40.4%（Acc6.12$\rightarrow$ 8.59）...
>- BRepCLIP-Score作为新的CAD感知生成评估指标，比图文原生的CLIP Score更敏感于结构错误。
 
>[!limitation]
>- **固定的语义分类体系**：face/egde的基元类型限于预定义分类(平面、柱面、直线、圆弧等)，无法覆盖真实工程数据的多样性；同时，所用数据集中几何结构以简单的拉伸旋转特征为主，**自由曲面**占比较小。
>- **固定几何分辨率**：face/edge tokenization的分辨率固定，对于复杂CAD模型可能不足，且计算/存储成本随着基元数量上升。
>- **没有利用参数方程信息**...


# 一些启示
1.**领域原生结构 > 通用表示**：对于CAD这种结构化数据，强行转换为点云/网格会丢失关键信息。应该像NLP对待token、CV对待像素一样。尊重领域的原生表示。

2.**评估指标的创新空间**：当现有指标（CLIP Score, CD）无法捕捉领域关键属性时，设计领域感知的评估指标本身就是重要贡献。

3.对于BRep信息利用仍有提升空间。
