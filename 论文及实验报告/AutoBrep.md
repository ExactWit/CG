AutoBrep 是 SIGGRAPH Asia 2025 的工作，由 Autodesk Research 提出，核心目标是**用单个自回归 Transformer 端到端直接生成高精度、水密的 B-Rep 实体**。它解决了此前 B-Rep 生成方法多阶段 cascade 导致的误差累积、复杂度受限、训练流程繁琐的问题，同时原生支持 B-Rep 自动补全的条件生成任务。

下面从「B-Rep 如何表示」和「B-Rep 如何生成」两个核心维度展开。

---

## 一、B-Rep 的统一离散表示（核心创新）
AutoBrep 最关键的设计，是把 B-Rep 中异构的**几何信息**（曲面、曲线坐标）和**拓扑信息**（面-边邻接关系），全部编码为一条统一的离散 token 序列，从而可以直接用 GPT 式的下一词预测来建模。

### 1. 几何表示：从参数曲面到离散几何 token
几何部分的表示建立在「点格（Point Grid）」的统一抽象之上，再通过深度学习压缩为离散 token。

#### （1）底层统一表示：参数域点格
继承自 BrepGen 的思路：不对平面、圆柱、自由曲面等不同类型的曲面做区分，而是**在每个面的参数域 [u_min, u_max]×[v_min, v_max] 上均匀采样 N×N 个三维点**，形成点格；同理，每条边在其曲线参数域上采样 N 个点，形成 1D 点格。
- 实验中 N=32，即每个面对应 32×32×3 的点张量，每条边对应 32×3 的点张量。
- 优势：所有几何实体共享同一种输入格式，模型无需感知底层曲面的数学类型，泛用性强。

**UV 原点不变性处理**：
相同的几何曲面可以有不同的参数化方式（比如原点在哪个角、u/v 方向是否翻转），会导致同一张面得到不同的点格。AutoBrep 没有用复杂的等变卷积，而是做了一个简单的归一化：
对点格的四个角点按 xyz 字典序排序，强制让排序最小的角点作为参数域原点，通过翻转点格实现对齐。

#### （2）压缩编码：Deep Compression 自编码器
原始 32×32 的点格维度太高，直接 token 化会导致序列过长。AutoBrep 用 **Deep Compression Autoencoder** 对做点格压缩，相比 BrepGen 使用的 Stable Diffusion 式 VAE，压缩率更高且重建精度更好：
- 面：32×32×3 → 2×2×4 的隐层特征（共 4 个隐向量）
- 边：32×3 → 2×4 的隐层特征（共 2 个隐向量）

#### （3）离散化：FSQ 有限标量量化
为了适配自回归的离散 token 生成，AutoBrep 用 **FSQ（Finite Scalar Quantization）** 把连续隐向量量化为整数 token，替代传统的 VQ-VAE：
- 对 4 维隐向量，分别设置量化档位 [8, 5, 5, 5]，总码本大小为 8×5×5×5 = 1000。
- 相比 VQ，FSQ 无需码本查找、没有索引崩溃问题，码本利用率天然 100%，也不需要辅助损失，训练更简单。
- 训练自编码器仅使用点格重建的 MSE 损失，去掉了 VAE 的 KL 正则项。

#### （4）全局位置：边界框 token
因为点格会被归一化到 [-1, 1]³ 空间（保证平移缩放不变性），丢失了全局位置信息。AutoBrep 为每个面/边额外配对一组**边界框坐标 token**：
- 将整个实体的 [-1,1]³ 空间沿每个轴划分为 1024 个离散 bin，用 6 个离散 token 表示包围盒的对角点 [x₀, y₀, z₀, x₁, y₁, z₁]。
- 最终，「边界框 token C + 几何 token G」共同唯一确定一个面或边的几何与空间位置。

### 2. 拓扑表示：BFT 遍历 + 局部参考 token
B-Rep 的拓扑是面-边-顶点的图结构，要变成线性序列，需要解决两个问题：**按什么顺序排列面和边**、**如何编码面与边的邻接关系**。

#### （1）序列顺序：面邻接图的广度优先遍历（BFT）
AutoBrep 将 B-Rep 抽象为**面邻接图**：节点是 B-Rep 的面，边是两个面共享的 B-Rep 边。然后对这张图做**广度优先遍历（BFS）**，按遍历顺序把面和边展开为线性序列：
- 起始点：无条件生成时，选择实体上坐标最靠左下的面作为第 0 层起点。
- 逐层扩展：每一层新增的面，都是上一层面的未访问邻接面；连接新旧层的边、同层相邻面之间的边，在其连接的两个面都出现后立刻加入序列。
- 同层排序：同一层的多个面按包围盒坐标升序排列，保证顺序稳定。
- 哨兵 token：用特殊 token 标记序列起止、层级起止、单个面的起止。

这种顺序的优势：生成过程是「从一个基准面开始，一圈一圈向外长出相邻面」，符合实体的生长逻辑，也天然限制了拓扑关联的范围。

#### （2）邻接关系：局部拓扑参考 token
要表示一条边连接哪两个面，不需要全局面 ID——由 BFT 的性质可知：**第 l 层的边，只能连接到第 l-1 层的面，或者同层中已经出现的面**。
基于此，AutoBrep 设计了**滑动局部上下文窗口**（只包含最近两个 BFT 层级）：
- 在窗口内给每个面分配局部参考编号 T₀, T₁, T₂ ...
- 每条边只需要 1 个拓扑 token：因为边紧接在它的第二个连接面之后，第一个连接面是默认已知的，token 只需指向另一个连接面的局部编号。
- 当遍历推进到下一层级时，窗口滑动，局部编号重置。

论文消融实验验证：这种局部参考的效果显著优于全局面 ID；而如果按全局坐标排序面、不用 BFT，效果会大幅下降。

## 二、B-Rep 的自回归生成机制
有了统一 token 表示后，B-Rep 生成就变成了标准的自回归序列生成任务。

### 1. 模型架构
标准 GPT 风格的解码器 Transformer，配置如下：
- 16 层自注意力，32 个注意力头，隐层维度 2048，上下文长度 3072 token
- 工程优化：Swish-GLU 前馈、RMSNorm、RoPE 旋转位置编码、Flash Attention、分组查询注意力（GQA）
- 支持生成最多 100 个面、1000 条边的复杂实体

### 2. 无条件预训练
- 训练数据：ABC-1M（约 130 万单实体 B-Rep），按面数分层抽样保证复杂度均衡。
- 训练目标：标准下一词预测，交叉熵损失。
- 关键训练技巧：
  1.  **随机旋转增广**：对实体做 90°/180°/270° 旋转，改变 BFT 的起始面和遍历顺序，避免模型过拟合到固定序列。
  2.  **层级注意力 Dropout**：预测第 l 层 token 时，随机屏蔽 l-2 层及更早的 token，强迫模型依赖局部拓扑结构，提升泛化性。
  3.  **复杂度元 token**：在序列开头加入 Easy/Medium/Hard/Random 元 token，可在推理时控制生成实体的复杂度。

### 3. 推理与后处理
1.  **采样生成**：使用 top-p=0.9、温度 1.0 采样，半精度 + KV 缓存加速推理。
2.  **序列解码**：
    - 从 token 序列中解析出每个面/边的边界框和几何 token；
    - 几何 token 通过自编码器解码器还原为点格，再拟合为 B 样条曲面/曲线；
    - 拓扑参考 token 解码为面-边邻接关系。
3.  **实体缝合**：用 OpenCascade 等几何内核将曲面、边按拓扑关系缝合成水密的 B-Rep 实体，最终输出 STEP 格式。

### 4. 原生条件生成：B-Rep 自动补全
AutoBrep 不需要修改模型结构，仅通过前缀注入即可实现**局部面约束的 B-Rep 补全**，且能严格保证输入面的几何完全不变：
1.  将用户提供的部分面与边作为 BFT 的第 0 层，编码为 token 前缀，放在序列最前端。
2.  输入面中未闭合的悬空边，用特殊的 `T_u` 占位 token 标记，表示另一端连接的面待生成。
3.  模型从前缀末尾开始自回归续写后续层级，训练时损失仅计算生成部分，因此输入面不会被修改。
4.  训练时随机选择实体上的面作为「用户输入」做微调，70% 选装配界面（螺栓孔、凸包平面），30% 选随机面；同时加入最多 15% 的平移缩放增广，适配真实输入的边界框差异。

## 三、核心优势与结论
1.  **质量与水密性更优**：在 ABC 数据集上，有效水密率 70.8%，覆盖率 71.49%，全面优于 BrepGen、HoLa 等多阶段扩散基线。
2.  **复杂度与扩展性强**：基线方法通常最多支持 50 个面，AutoBrep 可支持 100 面以上的复杂实体，且面数增加时有效率下降更平缓。
3.  **推理速度快**：单张 A10G 上平均每个面 0.46 秒，比扩散方法快 2~5 倍，得益于自回归 + KV 缓存的高效推理。
4.  **训练流程简洁**：从多模块 cascade 简化为单模型预训练 + 少量微调，模块间没有误差累积。

## 四、局限
- 窄空间内密集面容易出现自交；
- 螺纹等细长缠绕的曲面、薄壳零件的小面，受点格分辨率和量化精度限制，生成效果较差；
- 仅支持局部面补全，不原生支持文本、图像等跨模态条件生成。
